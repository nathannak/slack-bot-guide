Building/Automating task with Bots

Overview

Slack, a cloud based team collaborations tool is no longer news to several people and teams all over the world. Since its launch in August 2013, several companies embraced Slack because of the ease with which it helps their team members to collaborate on projects both on site and offsite(remote). Much more than that, slack has several integrations that could make teams work smart with its bots integration. This simple means that you could use the bot to do whatever you want to do or use it to enhance tasks.


What is a bot?

In simple terms, a Bot is software that is designed to automate different types of task that we would have to do in our day to day activities.

Despite the fact that bots helps us to do several things at a time that we would have been able to do ourself, it could means different things to different people. E.g Gamers understand bots as AI characters in a game, spam messages for cyber criminals/web attackers, team collaboration for project manages, assistants to several customer service agents etc.

Of a truth, Bots are now an essential part of our lives as they make life easy for people generally.


What can I do with bots?

There is no doubt bots can do limitless tasks and automations. Out this numerous things that bots can do, here are a few, grouped by types:

  • Integration with other services.
  • Meeting Scheduler and reminder
  • Games
  • Customized notifications
  • Social services

Getting Started

To get started with simple custom Slack bot or a more complex bot that integrate with other services and put together those informations in a specified channel on slack, customized notifications, social services and meeting scheduler etc, which allows you to receive events in real time and send messages to channels, private groups and users.

Here are the basic steps that you need to follow:

1. Create a slack team if you don’t have one or belong to an organization

Creating a slack team is simple — go to the slack create team page, enter your email and follow the instructions.

This will take you to a page where you have to confirm your email address. Once your team is created, let’s jump to the next step.

Note: If you do have a team already, make sure you are logged in and jump to the next step.

2. Create a Hubot Slack Token

Generally, because slack bots don’t have password, you will have to create or generate a token for the bot.

To get this done, on the slack window, click on the team name and click Apps and Custom Integrations.

.
After this, click on apps and integration.

This will take you to slack apps page. On this page, click on bots under categories. Search for hubots and click on it. This will further take you to a Hubot page

click install and follow the next instructions.

Setting up the project

Sample Hubot project:

To create a hubot project, Yeoman is one of the best tools for getting started on your first Hubot. To install Yeoman, run 

npm install -g yo generator-hubot
This will install the Yeoman Hubot generator. After this, you can then create a new Hubot project using the generator. In this case, let’s call it notifier by running 
mkdir notifier && cd $_ 
yo hubot --adapter=slack

This script will prompt you enter values for certain attribute of the app we are building, and create a package.json file with which npm needs to help manage your project.

Now you got the bot(notifier) token generated and also a scaffolded hubot project, let run the project with the token generated 

HUBOT_SLACK_TOKEN=xoxb-184645738727-rxUvH3FSI1SW7XuKRwRrK8lT ./bin/hubot --adapter slack
This will immediately activate the notifier bot created and could see that the bot is now active on slack

Basic bot commands and usage

First and foremost, create a file in the script folder and name it sample.js or sample.coffee if you want to use coffeescript. Bots generally can be programmed to listen to conversations. This liStening can either be as a direct message or in a conversation.

In direct message, use .respond 


'use strict'; 
 module.exports = function(bot) { 
   bot.respond(/hola/i, function(res) { 
     return res.reply('Hola, que tal?'); 
   }); 
 };
This will respond to direct messages to the bot alone

Listening in any conversation - .hear


bot.hear(/\bbot\b/, function(res) { 
   return res.reply('I\'m sleeping'); 
});

Bots can talk.

I mean it sounds funny, but bots can talk. They respond to the listened conversations whenever the conversation concerns them by either send or reply. In the previous examples, we have been using reply. This will talk back to the user that is engaging the bot in the channel.


bot.hear(/\bhappy\b/, function(res) {
   return res.send('Yes :)');
});

The bot have brain which is used to store information via the a persistence(Redis) memory.

Note If Redis is not running, the bot will still function, but all data is lost next time the bot restarts. 


bot.hear(/\bcount\b/, function(res) {
   robot.brain.set('countValue', (robot.brain.get(countValue) || 0) + 1);
   return res.send(robot.brain.get(countValue));
});

Deploying to Heroku

Heroku is a cloud platform that lets companies build, deliver, monitor and scale apps.
With Heroku you can easily deploy your app with few clicks or few commands.

Here are few steps to deploying our newly created hubot

  • Create an account
  • Download the heroku toolbelt and install,
  • On Your terminal, Create the application by running e.g 
    heroku create notifierbot
  • Add the files with by running 
    git add .
  • Commit the files 
    git commit -am "intial commit"
  • Push to heroku 
    git push heroku master
  • Add the add-ons 
    heroku config:set HUBOT_SLACK_TOKEN=xoxb-184645738727-rxUvH3FSI1SW7XuKRwRrK8lT
  • Keep the bot awake 
    heroku config:set HUBOT_HEROKU_KEEPALIVE_URL=https://gmt-bot.herokuapp.com --app gmt-bot
Note: Yoeman should create a Procfile in the application root folder, but you can't find there, add a script to run the app in the package.json such as 
"scripts":{
  "start": "bin/hubot -a slack"
}
